Example: Natural Numbers

{-@ type Nat = {v:Int | 0 <= v} @-} {-@ nats :: [Nat] @-} nats = [0, 1, 2, 3]

Refinement Types are Contracts

{-@ plus :: x:Nat -> y:Nat -> {v:Nat | v = x + y} @-} plus x y = x + y {-@ minus:: x:Nat -> y:Nat -> {v:Nat | v = x - y} @-} minus x y = x - y

Input Type is a Pre-Condition

Specifies the input values must be non-negative (for illustration)

Output Type is a Post-Condition

Specifies the output value equals the sum/difference of the inputs

Automatically proved by SMT Solver


How about proofs about user-defined functions?

Beyond automatic SMT, but user can write proofs as programs

Theorems about Functions


{-@ reflect sum @-} {-@ sum :: n:Nat -> Int @-} sum 0 = 0 sum n = n + sum (n-1)

How can we prove the the theorems

\(\mathit{sum}(1) = 1\),

\(\mathit{sum}(2) = 3\),

\(\mathit{sum}(3) = 6\).

Refinement Reflection


The annotation {-@ reflect sum @-} generates the postcondition

sum :: n:_ -> {v:_ |v = if n == 0 then 0 else n + sum (n-1)}


But sum is uninterpreted function

SMT does not "understand" its definition

Reflect Function into Output Type


The type of sum connects implementation and specification

By converting implementation into postcondition

sum :: n:_ -> {v:_ |v = if n == 0 then 0 else n + sum (n-1)}


Key Idea

Calling sum n reveals definition of sum at n to SMT

Reflection at Result Type


Lets prove sum 3 = 6

{-@ sum3 :: _ -> { sum 3 == 6 } @-} sum3 _ = [ sum 0 -- :: {sum 0 = 0 } , sum 1 -- :: {sum 1 = 1 + sum 0} , sum 2 -- :: {sum 2 = 2 + sum 1} , sum 3 -- :: {sum 3 = 3 + sum 2} ] -- SMT connects the dots.

Removing any of the above "breaks" the chain of equalities, yields type error!

Structuring Proofs as Calculations


Haskell lets you define nice infix operators e.g. 

(===) :: x:a -> y:{a | y == x} -> {v:a|v == x && v == y}

Require both sides are equal

Ensure result is equal to both

Structuring Proofs as Calculations


The above proof becomes a "calculation"


{-@ sum3' :: _ -> { sum 3 = 6 } @-} sum3' _ = sum 3 === 3 + sum 2 === 3 + 2 + sum 1 === 3 + 2 + 1 + sum 0 === 6

Reusing Proofs: Functions as Lemmas

Proofs are functions

Reuse by calling the function

{-@ sum4 :: _ -> { sum 4 = 10 } @-} sum4 _ = sum 4 === 4 + sum 3 ? sum3 () -- "tell" typechecker: sum 3 == 6 === 10

? is a library operator (read ``because'')

Adds { sum 3 == 6 } to the context of known facts
